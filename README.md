# 画像分割ツール

縦長の画像を自動で 2000×900px（幅900 × 縦2000）のタイルに分割するシンプルなWebアプリです。ブラウザのみで処理が完結するため、サーバー不要です。

## 📋 仕様

### 入力形式
- **対応形式**: PNG、JPEG、WebP、GIF など（ブラウザが対応する画像形式全般）
- **サイズ制限**: ブラウザのメモリに依存（通常は数十MB程度まで対応）

### タイルサイズ
- **標準タイルサイズ**: 幅900px × 縦2000px
- **出力フォーマット**: PNG / JPEG から選択可能

### 処理仕様
- 縦長画像を上から下に向かって、2000px単位で分割
- 最後のタイルが2000pxに満たない場合はそのサイズで出力
- 画像の幅が900pxと異なる場合：
  - **900px未満**: 900pxに引き延ばし
  - **900px以上**: 900pxに縮小

## 🚀 使い方

### ステップ1: アプリを開く
[こちら](https://ueda-recore.github.io/image-splitter/) をクリックしてアプリを開きます

### ステップ2: 画像をアップロード
以下のいずれかの方法で画像をアップロード：
- **ドラッグ&ドロップ**: アップロード領域に画像をドラッグして放す
- **クリック選択**: アップロード領域をクリックしてファイルを選択

### ステップ3: プレビューで確認
- 元の画像サイズが表示されます
- 分割されたタイルが一覧で表示されます
- 各タイルのサイズが表示されます

### ステップ4: フォーマットを選択
- PNG（推奨）：高品質、ファイルサイズ大
- JPEG：圧縮、ファイルサイズ小

### ステップ5: ダウンロード
- **個別ダウンロード**: 各タイルの「ダウンロード」ボタンをクリック
- **一括ダウンロード**: 「すべてダウンロード」ボタンをクリック（300ms間隔で順次ダウンロード）

## 🔄 動作フロー

```
画像アップロード
    ↓
HTML5 FileReader で読み込み
    ↓
Canvas に描画して寸法確認
    ↓
TILE_HEIGHT（2000px）単位で分割
    ↓
各タイルを Canvas に描画
    ↓
幅を900pxに統一（拡大/縮小）
    ↓
PNG/JPEG に変換
    ↓
プレビュー表示 & ダウンロード可能に
```

## 💻 技術仕様

### 使用技術
- **HTML5 Canvas API**: 画像の分割・リサイズ処理
- **Vanilla JavaScript**: ES6+（フレームワーク不使用）
- **クライアントサイド処理**: サーバー不要、ブラウザのみで完結

### 主要な処理
1. **画像読み込み**: `FileReader API` で画像をデータURL化
2. **メモリ処理**: 元画像を Canvas に描画して自動スケーリング
3. **分割処理**: `Math.ceil(height / TILE_HEIGHT)` でタイル数を計算
4. **リサイズ処理**: `Canvas.drawImage()` で幅900pxに統一
5. **出力処理**: `Canvas.toBlob()` で PNG/JPEG に変換、`URL.createObjectURL()` でダウンロード

### 環境
- 対応ブラウザ: Chrome、Firefox、Safari、Edge など（Canvas API対応ブラウザ）
- 処理速度: 数秒（画像サイズと解像度に依存）

## 💡 使用例

### 例1: 縦2000pxの画像
- 入力: 幅1200px × 縦4500px の画像
- 処理:
  - 幅1200px → 900px に縮小
  - 高さ4500px を2000px単位で分割
  - 出力: 2個のタイル（900×2000px）+ 1個のタイル（900×500px）

### 例2: 縦2000pxより小さい画像
- 入力: 幅500px × 縦1500px の画像
- 処理:
  - 幅500px → 900px に引き延ばし
  - 高さ1500px はそのまま1つのタイル
  - 出力: 1個のタイル（900×1500px）

## 📝 注意事項

- **データ保護**: すべての処理がブラウザ内で完結するため、画像がサーバーに送信されることはありません
- **ブラウザ互換性**: Canvas API対応ブラウザが必要です（ほぼすべての最新ブラウザで対応）
- **メモリ制限**: 極端に大きな画像（5000×5000px以上）は処理が遅くなる可能性があります
